# Reviewer UX / UI Specification

## Goal
Provide a reviewer experience that is precise, efficient, and appropriate for multimedia artistic research.

## Best-Practice Model
The review workspace combines:
- HTML manuscript reader
- structured review form
- side-panel annotations
- optional PDF export
- access to source files where permitted

## Layout
### Header
- manuscript title
- article type
- deadline
- review status
- conflict reminder
- save draft / submit actions

### Left navigation
- section outline
- figures / tables / media index
- unresolved annotations
- quick jump to major sections

### Main pane
- HTML review edition
- numbered paragraphs
- figure numbers
- table numbers
- stable anchors
- inline media playback
- transcript access

### Right pane
- annotation thread panel
- filter by block type
- filter by unresolved / resolved
- add new annotation

### Review form tab
- expertise level
- summary
- strengths
- major issues
- minor issues
- ethical concerns
- comments to editor
- recommendation

## Annotation Targets
- paragraph
- heading
- figure
- table
- equation
- bibliography item
- video time range
- audio time range

## Required Submission Rules
Reviewer must provide:
- summary
- at least one strengths field entry
- at least one recommendation
- comments to author
- comments to editor optional but encouraged

## Save / Draft Behavior
- autosave every 30 seconds
- explicit save button
- last saved timestamp visible

## Accessibility
- keyboard navigation
- readable focus states
- captions / transcripts for media
- scalable text
